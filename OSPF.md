选择动态路由时，主要看的方面是选路佳，收敛快，占用资源少，RIP都不符合，所以我们来看OSPF



OSPF————开放式最短路径优先协议

1，OSPF传递的是拓扑信息，之后，再使用SPF算法计算到达未知网段的路由信息，至少保证算出来的路径没有环路ospf是使用带宽作为开销值的评判标准。所以，选路角度优于RIP

2，因为OSPF的计时器时间更短，所以，OSPF的收敛速度快于RIP的。

3，从单个数据包的角度来看，因为OSPF携带的是拓扑信息，所以，其资源占用大于RIP；从整体的角度看，因为OSPF并不存在像RIP这样高频的周期更新，所以，资源占用略优于RIP;



RIP三个版本：

​	RIPV1，RIPV2————IPv4

​	RIPNG————IPv6



OSPF三个版本：

​	OSPFV1（实验室阶段天折），OSPFV2————IPV4

​	OSPFV3————IPV6



RIPV2和OSPFV2相同点：
	1，OSPFV2和RIPV2一样都是无类别的路由协议；

​	2，OSPFV2（224.0.0.5（小弟的组播i地址）和224.0.0.6（大哥的组播地址））和RIPV2一样都是通过组播的形式传递信息；

​	3，OSPFV2和RIPV一样都支持等开销负载均衡；

以及：RIP协议只能应用在小型网络环境中，OSPF协议可以应用在中大型网络环境中（主要是因为OSPF协议可以实现结构化部署，其实就是区域划分）。 

区域划分的目的————区域内部传递拓扑信息，区域之间传递路由信息————也就是链路状态型协议的距离矢量特征

区域边界设备——ABR同时属于多个区域，一个接口对应一个区域，并且ABR至少有个接口在区域0中



区域划分的要求：
	1，区域之间必须存在ABR设备

​	2，必须按照星型拓扑进行划分---我们将中间这个区域称为骨干区域---骨干区域的区域ID必须为0(area id——32位二进制构成——点分十进制，直接使用十进制表示)







## 1，OSPF的数据包

**1，hello包**————周期性的发现，建立以及保活邻居关系

​	hello时间---10s（默认）（10S或者30S)

​	dead time---4倍的hello时间





​	RID（这东西是为了给区域中的每个路由器一个号码，方便认识）（为什么不直接给IP地址呢，因为路由器有很多接口呀） 

​		1，全网唯一；

​		2，格式统一，统一使用IP地址的格式--32位二进制构成，使用点分十进制表示

​	如何配置RID？	

​		1，手工配置

​		2，自动生成---如果设备存在环回接口，则使用环回接口中IP地址最大的作为RID；如果设备没有环回接口，则使用物理接口中IP地址最大的作为RID

**2，DBD包**————数据库描述报文（携带的是我有哪些拓扑信息的目录，并非LSA）

​	这就不得不提LSDB了---链路状态数据库---存放LSA的数据库（LSA就是那个拓扑结构的信息，为什么要存放呢，因为要等全部收集完了之后再画图）

​	相当于别人给我发目录，我先对照遍，看看自己有没有

**3，LSR包**————链路状态请求报文————基于DBD包请求未知的LSA信息。



**4，LSU包**————链路状态新报文————真正携带LSA的数据包



**5，LSACK包**————链路状态的确认报文————确认包



另外，OSPF其实存在每30MIN一次的周期更新

**总结：**

**Down状态**---发送hello包后，进入到下一个状态

**init状态（初始化状态）**---当收到其他设备发送的hello包中存在自己本地的RID时，进入到下一个状态

**Tow-way（双向通信）状态**--标志着邻居关系的建立

（条件匹配）--条件匹配成功，则进入之后的状态；条件匹配失败，则将保持邻居关系，周期性发送hello包进行保活即可。

**Exstart（预启动）状态**---使用未携带数据的DBD包进行主从关系选举，为主的可以优先获取LSA信息

**Exchange（准交换）状态**---发送携带数据数据的DBD包共享数据库的摘要信息

**Loading（加载）状态**---使用LSR/LSU/LSACK基于DBD包获取未知的LSA信息

**FULL（转发）状态**--标志着邻接关系的建立

------------

## 3，OSPF的工作过程

启动配置完成后，OSPF向本地所有运行协议的接口以组播224.0.0.5的形式发送hello包。hello包中包含本地的RID以及本地已知邻居的RID，之后，建立邻居关系---**邻居表**。

邻居关系建立完成后，则进行条件匹配，失败则停留在邻居关系，仅使用hello包进行周期保活即可。匹配成功，则开始建立邻接关系。首先，使用未携带数据的DBD包进行主从关系选举，之后，使用携带数据的DBD包共享数据库的目录。之后基于DBD包中未知的LSA，使用 LSR/LSU/LSACK报文获取。将所有LSA信息放在数据库中，生成**数据库表**。

最后，基于收集到的LSA信息，生成有向图和最短路径树，计算出到达未知网段的路由信息，添加到**路由表**中。
收敛完成后，依然每隔10s发送一次hello包进行周期保活，每隔30min进行一次周期更新。



结构突变，如：
	1，突然新增一个网段————触发更新，第一时间将更新信息通过LSU包传递，需要LSACK确认

​	2，突然断开一个网段————触发更新，第一时间将更新信息通过LSU包传递，需要LSACK确认

​	3，无法沟通（死亡时间会解决它的）

## 5，还有条件匹配的事：

![image-20260718031829908](C:\Users\xgz24\AppData\Roaming\Typora\typora-user-images\image-20260718031829908.png)

为什么要分出邻接和邻居呢，如图所示：

​	结构突变时，只有主要的那个路由器来对自己的小弟传输，小弟之间不会互相传输干扰，减少占用资源

​	**还有，大哥的组播地址是224.0.0.6，小弟的组播地址是224.0.0.5，结构突变时就像224.0.0.6传就好了，大哥要给小弟们传时就用224.0.0.5**

​	指定路由器--DR---在一个广播域中，指定路由器和其他设备之间建立邻接关系。

​	备份指定路由器---BDR---在一个广播域中，备份指定路由器和其他设备之间也建立邻接关系。

​	注意：DR和BDR虽然名字中包含路由器，实际上，是接口的概念。



条件匹配---在一个广播域中，如果所有的设备都建立令接关系，则会出现大量的重复更新；所以，需要进行DR/BDR的选举留邻居关系即可。





DR和BDR的选举规则：

1，先比较接口的优先级，接口优先级最大的为DR。之后，再剩余设备里选择BDR

接口优先级默认为1

优先级可以手工修改，方便人为干涉选举

优先级：8位二进制构成---0-255

当将一个接口的优先级修改为0，则该接口将放弃DR和BDR的选举。

```
[r1-GigabitEthernet0/0/01ospf dr-priority ?

修改
```

2，如果接口优先级相同，则比较RID，RID大的设备所对应的接口为DR，之后选举BDR设备。



OSPF的选举是非抢占模式的- 选举时间等于死亡时间 （后续加入进来的路由器不会影响原本的DR和RID）











```\
<rl>reset ospf 1 process

重启1的进程
```





## 6，OSPF的拓展配置

1，OSPF的手工认证

```
[r1-GigabitEthernet0/0/01ospf authentication-mode md5 1 cipher 123456

对面也得改哟
```



2，OSPF的区域汇总

```
[r2-ospf-1-area-0.0.0.01abr-summary 192.168.0.0 255.255.252.0
area 0里加汇总网段，后面是汇总网段
```



3，加快收敛——减少计时器时间

 ```
 [rl-GiqabitEthernet0/0/0]ospf timer hello 5
 
 [rl-GiqabitEthernet0/0/0]ospf timer dead 40
 改去吧
 ```

​	注意：hello时间邻居之间必须相同，否则将无法建立邻居关系

​	注意：hello时间一旦修改，死亡时间自动按照4倍关系进行匹配

​	注意：修改死亡时间，hello时间不会变化

​	注意：注意死亡时间不同，邻居关系也无法建立

4，OSPF的沉默接口——如果将一个接口配置成为沉默接口，则这个接口将只接受，不发送OSPF的数据包

```
[r1-ospf-1]silent-interface GigabitEthernet 0/0/2
```



5，缺省路由

```
[r3-ospf-l]default-route-advertise
```

不过要求边界设备本身存在缺省

```
[r3-ospf-1]default-route-advertise always 
强制下发缺省(一敲这个，所有路由器都会有)
```
