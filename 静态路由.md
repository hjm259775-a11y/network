

路由表



路由器的转发原理--- 数据来到路由器上，路由器会根据数据包中的目标IP地址查看自己本地的路由表，如果路由表中存在记录，则将无条件按照记录转发；如果路由表中不存在记录，则路由器将丢弃该数据包。

Destination/Mask-目标网段

Proto---协议：Direct-直连路由---描述直连网段的路由---路由器接口直接相连生成的网段就是直连网段

​	  直连路由默认存在条件——--1。接口双UP；2。接口需要配置IP地址

Pre-优先级---当到达同一个目标网段存在多条路径时，则比较优先级，仅加载优先级最高的到路由表中。（数值越大优先级越低）

NextHop---下一跳---**到达目标网段下一个经过的路由器的入接口的IP地址**（直连除外，直连会指向自己出的接口）

Interface——出接口——从哪个接口发出

如：192.168.1.2找192.168.2.2

​	给路由器发：SIP：192.168.1.2，DIP：192.168.2.2，SMAC：192.168.1.2的mac地址，DMAC：192.168.1.1网关接口的mac地址

​	路由器会根据路由表看到192.168.2.2，就会去找192.168.2.1





路由器之间的连线---骨干链路---一般不添加额外的设备

用以下两种办法获取未知网段的路由信息

静态路由：由网络管理员手工添加的路由条目

动态路由：所有路由器运行相同的路由协议，之后，所有路由器彼此之间沟通交流，最终计算出到达未知网段的路由信息。





**静态路由**：

![image-20260711180435617](C:\Users\xgz24\AppData\Roaming\Typora\typora-user-images\image-20260711180435617.png)

```
[r1]ip route-static 192.168.3.0 24 192.168.2.2

分别为 目标IP网段 和 到达目标网段下一个经过的路由器的入接口的IP地址
添加静态路由
```

需要配置到那个网段的最短路径，防止出现路由环路（IP协议加的数据包中的TTL值（存活）就是为了防止路由环路）



拓展配置

**1，负载均衡**---如果到达同一个目标网段存在多条开销相似的路径时，则可以将这些路程同时添加到路由表中，当数据来到设备上后，设备将会拆

分流量，同时走这些路径，起到叠加带宽的效果。







**2，环回接口（虚拟接口）**

```
[r7]interface LoopBack 0
[r7-LoopBack0]

创建虚拟接口（0~1023）
```

```
<r7>ping -a 192.168.1.1 12.0.0.1

指定源IP地址进行ping 
```



**3，手工汇总**

​	如果到达多条连续的子网时，则可以将这些网段进行汇总，之后，仅编写一条达到汇总网段的路由信息即可，达到减少路由表路由条目，增加

​	转发效率的目的。

​	可通过CIDR——无类域间路由对其进行汇总（详情见OSI七层.md）

​	如：192.168.1.0/24和192.168.2.0/24汇总成192.168.0.0/22

​	路由表的匹配原则---最长匹配原则(精准匹配原则，根据掩码的网络位长度，哪个长选哪个)



**4，路由黑洞**————-汇总网段中包含实际不存在的网段，导致数据有去无回，造成链路资源的浪费（所以合理的子网划分和汇总可以减少黑洞）

​	什么导致的？汇总时把一些不属于他的IP地址汇进去了



**5，缺省路由（默认路由）**————一条不限定目标的路由条目，当本地路由表中所有路由条目全部匹配后，再匹配缺省

​	路由黑洞一旦和缺省路由相遇，将百分之百出现路由环路（原本数据会因为路由黑洞而被丢弃，但现在因为有了缺省路由就一直循环）

```
[r7]ip route-static 0.0.0.0 0 12.0.0.2

配置缺省路由
```



**6，空接口路由**————用来防止路由黑洞和缺省路由相遇出现环路问题。

```
[r7]ip route-static 192.168.0.0 22 NULL 0

在黑洞路由添加一个汇总路由的指向null
```





**7，浮动静态路由**————通过修改路由条目的优先级起到备份的作用

```
[r9]ip route-static 192.168.2.0 24 192.168.2.1 preference 61

可以自定义优先级
```
